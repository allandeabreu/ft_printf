# ft_printf

## Description

`ft_printf` is a 42 School project that recreates part of the behavior of the original C `printf()` function.

The goal of this project is to understand variadic functions, formatted output, type handling, number conversion, and how to return the correct number of printed characters.

This project builds a static library called `libftprintf.a`, which contains the function `ft_printf()`.

The function prints formatted output to the standard output and returns the total number of printed characters.

## Function Prototype

```c
int ft_printf(const char *format, ...);
```

## Mandatory Conversions

This implementation supports the mandatory conversions required by the subject:

| Specifier | Description                                    |
| --------- | ---------------------------------------------- |
| `%c`      | Prints a single character                      |
| `%s`      | Prints a string                                |
| `%p`      | Prints a pointer address in hexadecimal format |
| `%d`      | Prints a signed decimal integer                |
| `%i`      | Prints a signed decimal integer                |
| `%u`      | Prints an unsigned decimal integer             |
| `%x`      | Prints a number in lowercase hexadecimal       |
| `%X`      | Prints a number in uppercase hexadecimal       |
| `%%`      | Prints a percent sign                          |

## Usage Example

Create a `main.c` file:

```c
#include "ft_printf.h"

int main(void)
{
    int count;

    count = ft_printf("Hello, %s!\n", "42");
    ft_printf("Printed characters: %d\n", count);

    ft_printf("Character: %c\n", 'A');
    ft_printf("String: %s\n", "Piscine");
    ft_printf("Pointer: %p\n", &count);
    ft_printf("Decimal: %d\n", -42);
    ft_printf("Integer: %i\n", 42);
    ft_printf("Unsigned: %u\n", 4294967295u);
    ft_printf("Hex lowercase: %x\n", 255);
    ft_printf("Hex uppercase: %X\n", 255);
    ft_printf("Percent sign: %%\n");

    return (0);
}
```

Compile with:

```bash
cc -Wall -Wextra -Werror main.c libftprintf.a
```

Run with:

```bash
./a.out
```

## Algorithm Explanation

The `ft_printf()` function receives a format string and a variable number of arguments.

The algorithm works by reading the format string character by character:

1. Start the variadic argument list with `va_start`.
2. Initialize a counter to store the number of printed characters.
3. Loop through each character of the format string.
4. If the current character is not `%`, print it directly using `write()`.
5. If the current character is `%`, move to the next character and check the conversion specifier.
6. Send the specifier and the argument list to `ft_check_format()`.
7. `ft_check_format()` calls the correct helper function according to the specifier.
8. Each helper function prints its value and returns the number of characters printed.
9. `ft_printf()` adds each return value to the total counter.
10. End the variadic argument list with `va_end`.
11. Return the total number of printed characters.

This approach keeps the code simple, readable, and easy to extend.

## Data Structure and Design Choices

This project does not require a complex data structure.

The main structure used is `va_list`, which allows access to a variable number of arguments.

The project is divided into small helper functions:

| Function          | Responsibility                                         |
| ----------------- | ------------------------------------------------------ |
| `ft_printf`       | Parses the format string and counts printed characters |
| `ft_check_format` | Chooses which helper function to call                  |
| `ft_put_c`        | Prints one character                                   |
| `ft_put_s`        | Prints a string                                        |
| `ft_put_p`        | Prints a pointer address                               |
| `ft_put_d_i`      | Prints signed decimal integers                         |
| `ft_put_u`        | Prints unsigned decimal integers                       |
| `ft_put_x`        | Prints hexadecimal numbers                             |

This design was chosen because it makes the code easier to understand, test, debug, and maintain.

Instead of placing all logic inside `ft_printf()`, each conversion has its own function. This follows a clean and modular approach.

## Variadic Functions

The most important concept in this project is the use of variadic functions.

A variadic function can receive a variable number of arguments.

In C, this is handled with `<stdarg.h>`:

```c
va_list args;
va_start(args, format);
va_arg(args, type);
va_end(args);
```

In this project:

* `va_start` initializes the argument list.
* `va_arg` retrieves the next argument with the correct type.
* `va_end` cleans up the argument list.

Example:

```c
ft_printf("Name: %s | Age: %d\n", "Allan", 29);
```

When `%s` is found, the function retrieves a `char *`.

When `%d` is found, the function retrieves an `int`.

## Conversion Handling

### `%c`

Prints a single character using `write()`.

### `%s`

Prints a string character by character.

If the string is `NULL`, it prints:

```txt
(null)
```

### `%p`

Prints a pointer address in hexadecimal format.

If the pointer is `NULL`, it prints:

```txt
(nil)
```

Otherwise, it prints the address with the prefix:

```txt
0x
```

### `%d` and `%i`

Print signed decimal integers.

The implementation handles negative numbers and includes a specific case for:

```txt
-2147483648
```

This is necessary because converting `INT_MIN` directly to positive would cause integer overflow.

### `%u`

Prints an unsigned integer in base 10.

### `%x` and `%X`

Print hexadecimal numbers.

* `%x` uses lowercase characters.
* `%X` uses uppercase characters.

### `%%`

Prints a single percent sign.

## Return Value

`ft_printf()` returns the total number of characters printed.

Example:

```c
int len;

len = ft_printf("Hello\n");
```

The return value is:

```txt
6
```

Because the output contains five letters plus the newline character.

## Limitations

This implementation covers the mandatory part of the project.

It does not implement the original `printf()` buffer management.

It also does not implement bonus flags, width, or precision.

Unsupported behavior outside the mandatory conversions is not guaranteed.
